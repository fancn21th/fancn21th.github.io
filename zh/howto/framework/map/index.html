提示

# 引子

提示

ECharts 基础教程在新窗口打开 中 作者 抱怨 读者 不善于阅读 API 文档.

个人认为 这本质是 心智模型 的问题. 大部分 读者 会 基于 惯性思维 去做判断 下结论.

在 React 官网有 thinking-in-react在新窗口打开 一文 解释如何 用 React 的思维方式去 开发 web 程序.

在 Vue3 官网中有 单文件组件在新窗口打开 一文 做了类似的解释.

然而 Echarts 官网 echarts 官网在新窗口打开 我并没有 看到 相关的文章.

我们来看看 最基础的 一个 Echarts 案例, 绘制一个简单的图表在新窗口打开. 注意高亮的 17-36 行

重点 Echarts 的 API 可以 幼稚地 来说就是 设置 一个 option在新窗口打开 对象.
绘制一个简单的图表 点击查看源代码
<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8" />
    <title>ECharts</title>
    <!-- 引入刚刚下载的 ECharts 文件 -->
    <script src="echarts.js"></script>
  </head>
  <body>
    <!-- 为 ECharts 准备一个定义了宽高的 DOM -->
    <div id="main" style="width: 600px;height:400px;"></div>
    <script type="text/javascript">
      // 基于准备好的dom，初始化echarts实例
      var myChart = echarts.init(document.getElementById("main"));

      // 指定图表的配置项和数据
      var option = {
        title: {
          text: "ECharts 入门示例",
        },
        tooltip: {},
        legend: {
          data: ["销量"],
        },
        xAxis: {
          data: ["衬衫", "羊毛衫", "雪纺衫", "裤子", "高跟鞋", "袜子"],
        },
        yAxis: {},
        series: [
          {
            name: "销量",
            type: "bar",
            data: [5, 20, 36, 10, 10, 20],
          },
        ],
      };

      // 使用刚指定的配置项和数据显示图表。
      myChart.setOption(option);
    </script>
  </body>
</html>

# option 的动态构造

不难想到 在任意一个时刻 地图的 option 大致会包括:

  • 若干个 geo在新窗口打开 控制 地图的绘制

  • 若干个 series在新窗口打开 控制 地图上的图表的绘制

  • dataset在新窗口打开 控制 地图上的图表依赖的数据

  • 其他 例如 legend, tooltip 等等

随着 用户的交互 这些 option 还会动态变化.

下面是 demo 程序中的 option. 生产环境的 option 会更加复杂, 代码行数 会成倍增加.

# 方案

问题复杂 必然就会 考虑如何 分而治之

# SOLID 原则

  • 依赖倒置
重点 把 option 作为 底层的基础操作. 将 绘制地图的业务需求 作为 高层的业务抽象.

# 编程范式

  • 函数式
重点 借鉴 reducers在新窗口打开 把上文 提到的 绘制地图的业务需求 作为 纯函数 来实现.
